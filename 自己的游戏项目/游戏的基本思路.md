# 一、游戏核心定位

| 项目   | 内容                                |
| ---- | --------------------------------- |
| 游戏类型 | Roguelike + 下落动作（类 Downwell）+ 轻社交 |
| 核心体验 | 轻紧张 + 成长感 + 情绪治愈                  |
| 差异点  | 降低操作压迫感 + 强叙事（病人）+ 玩家协助机制         |
| 核心循环 | 进入病人 → 下落战斗 → 收集 → 治愈 → 成长 → 解锁   |

---

# 二、核心玩法（重点拆解）

## 2.1 核心玩法循环（Game Loop）

|阶段|行为|系统|产出|
|---|---|---|---|
|1|选择/随机病人|病人生成系统|关卡种子|
|2|进入心灵世界|Roguelike关卡|战斗/探索|
|3|下落战斗|操作 + 技能 + 宠物|情绪碎片 / 道具|
|4|关卡推进|随机事件 / 分支|build变化|
|5|Boss战|阶段验证|存档点|
|6|治愈结算|表现评价系统|奖励 + 分支结局|
|7|成长|等级 / 收集|长期动力|

---

## 2.2 核心设计要点

### （1）“Downwell式下落”改造

|维度|原版特点|本项目改造|
|---|---|---|
|节奏|高压、快节奏|降低速度 + 增加缓冲区|
|容错|低容错|增加血量/恢复机制|
|存档|无|**Boss节点存档**|
|体验|操作驱动|操作 + 叙事驱动|

👉 目标：从“硬核动作”转向“轻动作 + 治愈体验”

---

### （2）Roguelike构建（核心可玩性来源）

|模块|设计|
|---|---|
|随机关卡|每个病人 = 独立关卡种子|
|Build系统|技能 / 道具 / 宠物组合|
|分支选择|不同心理路径（攻击/共情/逃避）|
|不可重复性|病人一次性（强化稀缺性）|

---

### （3）病人系统（核心创意）

|机制|说明|
|---|---|
|随机生成|每个病人独立背景 + 关卡|
|唯一性|治愈后不再出现|
|双结局|完美 / 非完美治愈|
|社交延伸|老玩家帮助新玩家|

👉 **关键设计价值：**

- 内容无限扩展（低成本生成）
- 强情绪绑定（记忆点）
- 为“玩家互助”提供动机

---

### （4）联机机制（弱同步设计）

|项目|内容|
|---|---|
|联机时机|**仅Boss存档点触发**|
|类型|协助型 / 异步协作|
|目标|降低开发成本 + 控制复杂度|
|玩家行为|老玩家进入新玩家世界协助|

👉 本质：**“轻社交，不破坏单机体验”**

---

### （5）宠物系统（辅助成长）

|功能|作用|
|---|---|
|战斗辅助|自动攻击 / Buff|
|Build补充|弥补玩家操作不足|
|收集驱动|宠物图鉴|

---

## 2.3 核心差异总结

|维度|传统Roguelike|本项目|
|---|---|---|
|驱动力|强挑战|情绪治愈 + 收集|
|失败惩罚|高|降低（存档）|
|内容来源|关卡|病人|
|社交|弱|玩家互助驱动|

---

# 三、奖励机制设计

## 3.1 即时奖励（短期爽感）

|类型|表现|作用|
|---|---|---|
|情绪碎片|光效 + 震动|即时反馈|
|下落收集|连续获取|操作爽感|

---

## 3.2 核心奖励（中期动力）

|来源|奖励|
|---|---|

|随机掉落|唯一道具（功能/消耗）|

---

|外观|角色进阶表现|
|技能|Build扩展|

---

|---|---|

|宠物|功能+收集|

---

|---|---|

---

|---|---|

|节奏|不逼肝|

# 四、留存与增长机制（重点策略）

## 4.1 核心留存抓手

|机制|本质|
|---|---|
|病人唯一性|稀缺内容|
|图鉴收集|Completion驱动|
|随机性|重玩价值|
|Build变化|策略探索|

---

## 4.2 社交驱动留存（关键亮点）

|机制|作用|
|---|---|
|老带新|延长生命周期|
|协助奖励|促进玩家互动|
|高阶玩家需求|寻找挑战/价值感|

👉 **关键逻辑闭环：**

- 老玩家“无内容” → 转向“帮助新玩家”
- 系统提供奖励 → 社交形成循环

---

## 4.3 情绪驱动留存（差异化）

|设计|作用|
|---|---|
|病人故事|情感连接|
|感谢信|成就感|
|心灵大厅|展示空间|

---

# 五、风险与建议（关键补充）

## 5.1 潜在风险

|问题|说明|
|---|---|
|病人耗尽|内容枯竭|
|联机薄弱|社交不足|
|Roguelike深度不足|可玩性下降|

---

## 5.2 优化建议

|方向|建议|
|---|---|
|内容扩展|程序生成病人模板|
|联机深化|排行榜 / 共斗Boss|
|Build深度|技能组合增强|
|结局系统|多分支结局|

---

# 六、总结（一句话核心）

**这是一个以“心理治愈”为叙事核心，将 Downwell 下落玩法轻量化，并通过“随机病人 + 玩家互助”构建长期留存的 Roguelike 游戏。**
